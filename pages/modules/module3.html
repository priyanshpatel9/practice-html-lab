Document Relative File Paths

Document relative file paths are paths that are relative to the current document. They are the most flexible type of file paths because they allow you to move files around without breaking the links.

For example, if you have a file in a folder called images and you want to link to it from a file in the root of your website, you would use the path ./images/logo.png.

The ./ at the beginning of the path tells the browser to look for the file in the current directory. If you wanted to link to a file in a folder above the current directory, you would use ../.

Example

  • In the example above, if you were in the about.html file and you wanted to link to the help.html file, you would use the path ./help.html. Because they're both in the same directory.
  • If you were in the about.html file and you wanted to link to the index.html file, you would use the path ../index.html. Because the index.html file is in the directory above the about.html file.
  • If you were in the about.html file and you wanted to link to the module.html file, you would use the path ./modules/module1.html. Because the module1.html file is in the assignments directory, which is in the directory that's at the same level as about.html file.
  • If you were in the module1.html and you wanted to link to the index.html file, you would use the path ../../index.html. Because the index.html file is in the root directory, which is two levels above the module1.html file.
